## 🥇 1º LUGAR: ENGENHARIA DE CONFIABILIDADE DE SITES (SRE) - 19/20

### 💡 POR QUE COMBINA COM VOCÊ:

Esta carreira é o ponto de encontro perfeito entre seu interesse em entender sistemas e seu foco em código. Como você já tem 4 anos em provedor de internet, você já entende a criticidade da disponibilidade. O SRE aplica conceitos de engenharia de software para resolver problemas de infraestrutura e operações.

### ⚖️ O QUE ESPERAR:

**VANTAGENS:**

* Aproveitamento máximo do seu conhecimento de redes e conectividade.
* Alta valorização no mercado para profissionais que automatizam infraestrutura via código.

**DESAFIOS:**

* Exige domínio profundo de automação e ferramentas de observabilidade.
* Lidar com a pressão de manter sistemas de larga escala no ar.

### 📈 MERCADO:

Extremamente aquecido para quem faz a transição do suporte/infra tradicional para o modelo *Infrastructure as Code* (IaC).

## 🥈 2º LUGAR: ENGENHARIA DE DEVOPS - 18/20

### 💡 POR QUE COMBINA COM VOCÊ:

O DevOps foca na integração entre o desenvolvimento (seu desejo de codar) e a infraestrutura (seu interesse principal). Sua bagagem em provedores facilita muito o entendimento de fluxos de dados e protocolos que são a base das esteiras de implantação modernas.

### ⚖️ O QUE ESPERAR:

**VANTAGENS:**

* Carreira central em quase todas as empresas de tecnologia modernas.
* Possibilidade de trabalhar com tecnologias de ponta como Docker e Kubernetes.

**DESAFIOS:**

* Necessidade de aprender múltiplas ferramentas de CI/CD simultaneamente.
* Requer uma mudança de mentalidade de "consertar" para "prever e automatizar".

### 📈 MERCADO:

Alta demanda em empresas de todos os portes, com oportunidades sólidas de trabalho remoto.

---

## 🥉 3º LUGAR: ARQUITETURA DE CLOUD - 16/20

### 💡 POR QUE COMBINA COM VOCÊ:

Se você gosta de entender como os sistemas se conectam de ponta a ponta, o Arquiteto de Cloud desenha essas estruturas. Sua experiência em suporte técnico te deu a visão prática do que dá errado, o que é um diferencial enorme na hora de projetar soluções resilientes na nuvem.

### ⚖️ O QUE ESPERAR:

**VANTAGENS:**

* Foco estratégico e visão macro de como os sistemas operam.
* Transição natural para quem já domina os fundamentos de hardware e redes.

**DESAFIOS:**

* Exige certificações específicas e constantes atualizações dos provedores (AWS, Azure, GCP).
* Menos foco em "mão no código" diário comparado a SRE, focando mais em desenho técnico.

### 📈 MERCADO:

Crescimento constante, especialmente em empresas que estão migrando data centers físicos para a nuvem.

# 📦 Plano de Estudos (Agent 2)

## 🧩 VISÃO DO DIA A DIA

Como é o trabalho de um(a) **Engenheiro(a) DevOps**:

* **Automação de Pipelines:** Criar e manter fluxos de CI/CD para que o código chegue ao ambiente de produção de forma segura e rápida.
* **Infraestrutura como Código (IaC):** Provisionar servidores, clusters e redes usando scripts (como Terraform) em vez de cliques manuais em consoles.
* **Monitoramento e Observabilidade:** Configurar painéis (Dashboards) e alertas para garantir que o sistema esteja saudável e performando bem.
* **Colaboração com Devs:** Apoiar os times de desenvolvimento na arquitetura de aplicações nativas em nuvem (Cloud Native) e resolução de gargalos.
* **Gestão de Incidentes:** Atuar na causa raiz de problemas de disponibilidade, aplicando correções automatizadas que evitem a repetição do erro.

## 🚀 PROJETO DE PORTFÓLIO

**PROJETO:** *The Automated Web Stack*

**O QUE FAZER:**
Criar um repositório que provisiona automaticamente uma aplicação web (API) em um cluster Kubernetes na nuvem, utilizando Terraform para a infra e GitHub Actions para o deploy automatizado.

**ENTREGÁVEIS:**

* Código da Infraestrutura em Terraform (Iac).
* Workflow do GitHub Actions (YAML) configurado.
* Documentação (README.md) com diagrama da arquitetura.

**CRITÉRIOS DE ACEITAÇÃO:**

* O deploy deve ser acionado automaticamente ao fazer um "push" na branch main.
* A infraestrutura deve ser capaz de ser destruída/criada com um único comando.
* A aplicação deve ser acessível via uma URL pública (Ingress/Load Balancer).

**DICA:** Como você tem base em Suporte, adicione logs detalhados e uma seção de "Troubleshooting" no seu README. Isso brilha os olhos dos recrutadores.

## 💬 ROTEIRO DE ENTREVISTAS

**PERGUNTA 1:** Qual a diferença entre Integração Contínua (CI) e Entrega Contínua (CD)?
**COMO RESPONDER:** CI foca na integração frequente do código e testes automatizados para garantir qualidade. CD automatiza a liberação desse código para ambientes de teste ou produção, reduzindo o erro humano no deploy.

**PERGUNTA 2:** Por que usar Terraform em vez de criar recursos manualmente?
**COMO RESPONDER:** Pela rastreabilidade, versionamento e repetibilidade. Com IaC, evitamos o "Drift" de configuração e conseguimos replicar ambientes idênticos em minutos sem erros manuais.

**PERGUNTA 3:** Como você protege segredos (senhas/chaves) em um pipeline?
**COMO RESPONDER:** Nunca deixo em texto claro no código. Utilizo variáveis de ambiente protegidas no GitHub Secrets ou integraria com um Vault (como HashiCorp Vault ou AWS Secrets Manager).

**PERGUNTA 4:** O que é um Container e qual a vantagem sobre VMs?
**COMO RESPONDER:** Containers compartilham o kernel do SO host, sendo muito mais leves e rápidos que VMs. Eles garantem que a aplicação rode exatamente igual em qualquer ambiente (dev, homologação e prod).

**PERGUNTA 5:** Como sua experiência em Infra/Suporte contribui para sua atuação em DevOps?
**COMO RESPONDER:** Minha base me permite entender o que acontece "debaixo do capô". Sei lidar com problemas de rede, DNS e permissões de SO, o que me dá uma vantagem enorme no debug de problemas complexos de containers.
